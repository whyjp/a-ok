워커 프로파일
workerctl profiles 가 관리하는 워커 실행 프로파일 목록. 각 프로파일은 워커가 어떤 루트에서 기동될지를 묶는 라벨이다.
#이름기본 루트 생성업데이트
Hermes 스폰 세션
worker-control 이 직접 띄운 (= Hermes 오케스트레이션 대상) Claude 워커 세션. worker_sessions 테이블 기준. 모두 claude --permission-mode auto 일반 모드로 기동된다.
#이름상태runtime 프로파일프로젝트 tmux / pidstarted
Native Claude 세션
이 호스트의 ~/.claude/projects/ 아래에 남아있는 Claude CLI 세션 로그. 읽기 전용 으로 디스커버리한다 — 파일 위치/포맷이 호스트마다 다를 수 있으므로 휴리스틱으로 인코딩을 푼다.

관리 대상 프로젝트
workerctl projects scan 결과. owned_work (D:/work-github) 는 워커 기동/PR 가능, public_reference (D:/github) 는 read-only — 세션 시작이 정책으로 차단된다.
#이름워크스페이스정책 git브랜치경로마지막 스캔
정책: D:/work-github (owned_work, 편집/PR 가능) · D:/github (public_reference, read-only · 워커 기동 금지). 워커는 항상 claude --permission-mode auto 로 기동되며 claude -p 는 정책상 사용되지 않는다.
